# PlotIQ Design Reference

This file captures the operational visual guide provided as a design reference for PlotIQ.

## Core design direction

- **Regulatory intelligence infrastructure** rather than a consumer dashboard.
- **Procedural** and **institutional** rather than flashy or gamified.
- **Familiar formats**: feasibility reports, valuation reports, engineering documents, audit systems.
- **Trust artifact**: the PDF/report is the primary delivery surface and should feel citable, printable, and defensible.

## Visual system

- Primary background: `#F5F1EA`
- Border / separator: `#E3DCCF`
- Primary text: `#1A1A1A`
- Soft text: `#4A463F`
- Accent / signal: `#8B3A2A`
- Success: `#3F6B4A`
- Signal background: `#F0E6E1`

## Typography

- **Primary display text:** Source Serif 4
- **Mono / numerical text:** JetBrains Mono
- Use serif for narrative headings and calm institutional tone.
- Use mono for figures, regulatory references, and structured data.

## Structural principles

- Strong information hierarchy
- Clear sections with defined top padding and separators
- Calm whitespace and restrained card styling
- Grid-based layouts for data and token blocks
- Audit-style cards with plain labels, not flashy badges

## Report structure

The canonical report should include:

1. Context
2. Site summary
3. Regulatory basis
4. Existing conditions
5. Entitlement computation
6. Incentive logic
7. Feasibility metrics
8. Comparison layer
9. Assumptions
10. Traceability
11. Disclaimer
12. Provenance

## Interaction principles

- Prefer subtle motion and structured reveal
- Avoid startup-style microinteractions and animation
- Avoid AI theater, gamification, and disruptive language
- Prioritize calm, procedural behavior and evidence-first presentation

## Brand language

Use:
- entitlement estimate
- regulatory assessment
- feasibility analysis
- clause reference
- development potential
- compare offer
- assessment artifact
- regulatory intelligence

Avoid:
- AI-powered
- disruptive
- revolutionary
- smart engine
- next-gen proptech
- growth language
- startup jargon

## Next steps

Use this reference to:

- refine the landing page to feel more institutional
- design the app as a guided assessment workflow
- convert the report output toward a printable, audit-friendly artifact
- simplify UI interactions and remove dashboard-like language
